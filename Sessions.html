Sessions

Sessions allow us to track a user from page to page on the server. Each user/browser is assigned a unique identifier that is stored in a cookie. Every request, PHP reads the session cookie and regenerates the information stored on the server in the session variable. When the user closes the browser window their session information is lost.

Session hijacking is a common security concern.

<?php
// session_start() initialises the session system
// must be called before accessing the session info
session_start();

// with the $_SESSION super global we can persist info
// $_SESSION is an associative array
// the stored info is available every request, on every page
//   until the user quits the browser or we delete it
$_SESSION['page_views'] = 1;

// Use the session_destroy() function to stop using sessions
session_destroy();
