User Authentication

User authentication is the process of comparing credentials for validity. Often this process takes the form of comparing a username and password entered in an HTML form with a username and password stored in a database.

There are many security concerns with user authentication, including: securing passwords, session hijacking, securing cookies, injection attacks, XSS, etc.

Authentication Process Overview

  1. User fills in a username and password in an HTML form.
  2. User submits the form to a PHP script.
  3. The PHP script validates that a username and password were entered.
  4. The PHP script compares the username and password against those stored in a database.
  5. If the username and password match, PHP creates a session to remember the user.
  6. Every time a secure page is loaded, the session is checked to make sure it is still valid.

Password Security, Hashing & Salting

Passwords should always be stored in a database using a one-way encryption mechanism called hashing. The password is converted to a series of bits that are extremely difficult to turn back into the original password.

A password hash would look something like this: f7f8249717ab21078a82d61e46b8a8625e67b8b1

Salting is the process of adding some extra random pieces into the hash to make the password more unique and to throw off hackers. Adding a unique random salt to each password is very helpful in stopping rainbow table attacks.

The current best practice for hashing passwords is to use Bcrypt. And, if you’d like, with the addition of an extra shared application key and HMAC.

If your password is ever e-mailed to you that means the developer isn’t storing your password securely in the database.

Some user authentication hacking vectors include: rainbow tables, brute force, DoS attacks, session hijacking and many more.
